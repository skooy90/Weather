# 작업 지침

## 1. 파일 작업 전 확인사항
- 작업하기 전에 항상 기존에 파일이 있는지 확인
- 파일이 있다면 기존 데이터를 삭제하지 않고 사용
- 기존 변수명과 같은 것이 있는지 확인
  - 같은 변수명이 있다면:
    - 기능 수정이면: 기존 변수의 기능을 수정
    - 기능 추가면: 새로운 변수명으로 추가
- 기존 데이터의 손실이 없도록 주의

## 2. 코드 작성 전 확인사항
-프론트 서버는 ES모듈
-백엔드는 CommonJS문법
- 폴더 전체를 확인하여 MD 파일 존재 여부 확인
- 계획 문서 확인
- 이전 작업 내용 확인
- 변경 이력 확인
- 관련 문서들:
  - 앱_개발_계획.md
  - changes_summary.txt
  - 서버_작업_기록.md
  - 보안사항.md
  - 기술_스택.md
  
1. frontend/docs 디렉토리의 다음 문서들을 반드시 확인:
   - 메인 페이지 디자인 컨셉.md
   - 각 페이지별 꾸미기 계획.md
   - 프론트엔드_개발_작업_내용.md
   - 메인_페이지_컴포넌트_구현.md
   - 공용_변수명.md
   - 코드_작성_가이드라인.md
   - 관리자 페이지 디자인 컨셉.md
   - 메인 페이지 내용 계획.md
   - 관리자 시스템 구현.md
   - 웹 페이지 개발 계획.md
   - 웹 페이지 개발.md
   - 웹_서비스_실행_과정.md

2. 코드 스타일 가이드 준수:
   - ESLint 규칙 준수
   - Prettier 포맷팅 적용
   - 네이밍 컨벤션 준수

3. 컴포넌트 구조:
   - 재사용 가능한 컴포넌트는 components 디렉토리에 위치
   - 페이지별 컴포넌트는 pages 디렉토리에 위치
   - 스타일은 styled-components 사용

4. 상태 관리:
   - 전역 상태는 Context API 사용
   - 로컬 상태는 useState/useReducer 사용
   - API 호출은 custom hooks 사용

5. 에러 처리:
   - 모든 API 호출에 try-catch 적용
   - 사용자 친화적인 에러 메시지 표시
   - 로딩 상태 표시

6. 성능 최적화:
   - 불필요한 리렌더링 방지
   - 이미지 최적화
   - 코드 스플리팅 적용

7. 보안:
   - 민감한 정보는 환경 변수로 관리
   - XSS 방지
   - CSRF 토큰 사용

8. 테스트:
   - 주요 기능에 대한 테스트 코드 작성
   - 테스트 커버리지 80% 이상 유지

9. 문서화:
   - 주요 컴포넌트에 JSDoc 주석 추가
   - README.md 업데이트
   - API 문서 작성

10. 접근성:
    - ARIA 레이블 사용
    - 키보드 네비게이션 지원
    - 색상 대비 준수

## 3. 작업 순서
1. 프로젝트 구조 확인
2. 관련 문서 확인
3. 기존 코드 확인
4. 변경 계획 수립
5. 코드 수정/추가
6. 변경사항 검토
7. 문서 업데이트
8. 작업한 내용 "25일 작업내용.md"로 추가하기

## 4. 주의사항
- 기존 기능이 깨지지 않도록 주의
- 하위 호환성 유지
- 기존 코드를 사용하는 다른 컴포넌트들이 영향을 받지 않도록 주의
- 새로운 기능은 점진적으로 추가
- 문서화된 계획과 일치하는 개발 진행
- 보안 및 성능 관련 고려사항 준수
