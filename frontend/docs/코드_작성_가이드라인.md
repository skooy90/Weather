# 코드 작성 가이드라인
## 16. 주의사항
환경설정은 건드리지 않는다.
작업 내용은 
 - '(프론트,백엔드) 작업내용.md'에 작업내용을 정리한다.
우선 페이지부터 만든다.
- **도커 사용**
  - 서버 배포는 도커를 이용
  - 도커파일 및 설정 파일은 서버 경로 참고하여 제작
  - 컨테이너 설정 및 환경 변수 관리

- **문서 참고**
  - 모든 작업 시 관련 md 파일 참고
  - 계획, 설계, 개발 문서 확인
  - 변경사항 문서화

- **변수 관리**
  - 공용 변수명은 `공용_변수명.md` 파일에 관리
  - 변수 사용 시 주석으로 설명 추가
  - 변수명 충돌 방지

- **코드 주석**
  - 모든 코드에 상세한 주석 추가
  - 함수/메서드 설명
  - 복잡한 로직 설명
  - TODO/FIXME 표시
## 1. 기본 원칙
- 모든 코드는 한글로 작성
- 주석은 코드의 의도를 명확히 설명
- 일관된 코드 스타일 유지
- DRY(Don't Repeat Yourself) 원칙 준수
- 문서화는 마크다운(.md) 형식으로 작성

## 2. 프로젝트 구조
- 프론트엔드: `frontend/`
  - 컴포넌트: `components/`
  - 페이지: `pages/`
  - 스타일: `styles/`
  - 유틸리티: `utils/`
  - API: `api/`
  - 타입: `types/`
- 백엔드: `backend/`
  - 라우트: `routes/`
  - 컨트롤러: `controllers/`
  - 모델: `models/`
  - 미들웨어: `middleware/`
  - 설정: `config/`
- 문서: 루트 디렉토리
  - 계획/설계: `*_계획.md`, `*_디자인_컨셉.md`
  - 개발: `*_개발.md`
  - 기록: `작업_기록.md`, `changes_summary.txt`
  - 에러: `error_solutions.txt`, `에러_발생과_수정.txt`

## 3. 네이밍 컨벤션
- 컴포넌트: PascalCase (예: `MainHeader.tsx`)
- 함수/변수: camelCase (예: `handleClick`)
- 상수: UPPER_SNAKE_CASE (예: `MAX_COUNT`)
- 파일명: 
  - 코드: kebab-case (예: `main-header.tsx`)
  - 문서: snake_case (예: `메인_페이지_디자인_컨셉.md`)
- 클래스: PascalCase
- 인터페이스: PascalCase (접두사 'I' 사용)

## 4. 컴포넌트 작성
