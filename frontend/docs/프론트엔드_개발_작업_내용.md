# 프론트엔드 개발 작업 내용

## 1. 프로젝트 구조
### 1.1 디렉토리 구조
```
frontend/
├── src/
│   ├── components/
│   ├── pages/
│   ├── utils/
│   └── App.jsx
├── public/
└── docs/
```

### 1.2 주요 컴포넌트
- `App.jsx`: 메인 애플리케이션 컴포넌트
- `Header.jsx`: 헤더 컴포넌트
- `Home.jsx`: 메인 페이지
- `ContentDetail.jsx`: 콘텐츠 상세 페이지
- `Login.jsx`: 로그인 페이지
- `SignUp.jsx`: 회원가입 페이지

## 2. 라우팅 설정
### 2.1 라우터 구성
- `BrowserRouter`: 최상위 라우터 (main.jsx)
- 라우트 구성:
  - `/`: 메인 페이지
  - `/content/:id`: 콘텐츠 상세 페이지
  - `/login`: 로그인 페이지
  - `/signup`: 회원가입 페이지

### 2.2 라우팅 관련 기능
- `useNavigate`: 페이지 간 이동
- 동적 라우팅 파라미터 사용
- 보호된 라우트 구현

## 3. 상태 관리
### 3.1 Context API 사용
- `AuthContext`: 로그인 상태 관리
- `ThemeContext`: 다크모드 상태 관리

### 3.2 주요 상태
- `isLoggedIn`: 로그인 상태
- `isDarkMode`: 다크모드 상태
- `isSidebarOpen`: 사이드바 상태

## 4. 주요 기능
### 4.1 인증 기능
- 로그인/회원가입 구현
- 사용자 인증 상태 관리
- 보호된 라우트 구현

### 4.2 UI/UX 기능
- 다크모드 토글
- 반응형 사이드바
- 댓글 기능
- 콘텐츠 좋아요/공유

### 4.3 레이아웃
- 반응형 그리드 레이아웃
- 사이드바와 메인 콘텐츠 영역 분리
- 모바일 대응 (1024px 이하에서 단일 컬럼)

## 5. 스타일링
### 5.1 테마 설정
- 라이트 테마
- 다크 테마
- 테마 전환 애니메이션

### 5.2 컴포넌트 스타일
- 헤더 스타일
  - 고정 위치 (fixed)
  - 반투명 배경
  - 블러 효과
  - 그림자 효과
- 컨테이너
  - 최대 너비: 1200px
  - 중앙 정렬
  - 여백: 20px
- 콘텐츠 그리드
  - 반응형 그리드 (최소 300px)
  - 간격: 2rem
  - 하단 여백: 2rem
- 사이드바
  - 배경색: 테마에 따라 변경
  - 내부 여백: 1.5rem
  - 그림자 효과
  - 둥근 모서리 (0.5rem)

## 6. API 통신
### 6.1 API 함수
- `login`: 로그인 요청
- `signUp`: 회원가입 요청
- `getContent`: 콘텐츠 조회
- `addComment`: 댓글 추가

- 인증 에러 처리
