### 3. 크롤링 시스템
#### 3.1 데이터베이스 스키마
```sql
CREATE TABLE blog_posts (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    content TEXT,
    image_url TEXT,
    source_url VARCHAR(500) NOT NULL,
    author VARCHAR(100),
    category VARCHAR(50),
    created_at DATETIME,
    crawled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    affiliate_link VARCHAR(500),
    view_count INT DEFAULT 0,
    like_count INT DEFAULT 0,
    INDEX idx_category (category),
    INDEX idx_created_at (created_at),
    UNIQUE INDEX idx_source_url (source_url)
);
```

#### 3.4 크롤링 관리 시스템
##### 3.4.1 블로그 소스 관리
```sql
CREATE TABLE blog_sources (
    id INT AUTO_INCREMENT PRIMARY KEY,
    url VARCHAR(500) NOT NULL,
    name VARCHAR(100),
    type ENUM('manual', 'rss', 'keyword') NOT NULL,
    status ENUM('active', 'inactive') DEFAULT 'active',
    last_crawled_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE INDEX idx_url (url)
);

CREATE TABLE keywords (
    id INT AUTO_INCREMENT PRIMARY KEY,
    keyword VARCHAR(100) NOT NULL,
    category VARCHAR(50),
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

##### 3.4.2 크롤링 규칙
- **제외 조건**:
  - 제목/본문에 포함된 키워드:
    - '광고', '협찬', '이벤트', '공동구매'
  - 글자 수: 300자 이하
  - 이미지: 1장 이하
  - 카테고리: '공지', '이벤트', '제휴문의'
  - 협찬 문구 포함

- **크롤링 제한**:
  - 간격: 1~5초 랜덤
  - User-Agent: 모바일/데스크탑 브라우저 랜덤
  - 일일 크롤링 제한: 도메인당 30건

##### 3.4.3 크롤링 프로세스 상세
1. **초기 설정**:
   - 수동 등록 블로그 20~30개
   - 키워드 기반 검색 설정
   - RSS 피드 자동 등록 설정

2. **일일 크롤링 프로세스**:
   ```
   a. 수동 등록 블로그 크롤링
      - 최신 포스트 확인
      - 제외 조건 적용
      - 데이터 저장

   b. 키워드 기반 검색
      - 각 키워드별 검색
      - 신규 블로그 발견 시 자동 등록
      - 포스트 크롤링 및 저장

   c. RSS 피드 확인
      - 피드 업데이트 확인
      - 신규 포스트 크롤링
   ```

3. **IP 차단 방지 전략**:
   - 크롤링 간격 랜덤화
   - User-Agent 랜덤화
   - 일일 크롤링 제한
   - (필요시) 프록시 서버 도입

#### 3.7 RSS 피드 시스템
##### 3.7.1 피드 설정
```sql
CREATE TABLE rss_feeds (
    id INT AUTO_INCREMENT PRIMARY KEY,
    blog_id INT NOT NULL,
    feed_url VARCHAR(500) NOT NULL,
    update_interval INT DEFAULT 6, -- 시간 단위
    last_updated_at TIMESTAMP,
    status ENUM('active', 'inactive') DEFAULT 'active',
```
