5. 백엔드 구조

10. Render 서버 배포 과정
    - 프론트엔드 배포 (준비 완료):
      1. Render 대시보드에서 "New +" 클릭
      2. "Static Site" 선택 (정적 파일 서빙용)
      3. GitHub 저장소 연결
      4. 서비스 설정:
         * Name: weather-frontend
         * Environment: Node
         * Build Command: npm install && npm run build
         * Publish Directory: frontend/build
      5. 환경 변수 설정:
         * REACT_APP_API_URL=https://api.kysong.store
      6. "Create Static Site" 클릭

    - 백엔드 배포 (준비 완료):
      1. Render 대시보드에서 "New +" 클릭
      2. "Web Service" 선택
      3. GitHub 저장소 연결
      4. 서비스 설정:
         * Name: weather-backend
         * Environment: Node
         * Build Command: npm install
         * Start Command: npm start
         * Root Directory: backend
      5. 환경 변수 설정:
         * MONGODB_URI=보안된 연결 문자열
         * CORS_ORIGIN=https://kysong.store
         * JWT_SECRET=보안된 시크릿 키
      6. "Create Web Service" 클릭

    - 배포 후 확인 사항:
      1. 프론트엔드 서비스 상태 확인
      2. 백엔드 API 응답 확인
      3. MongoDB 연결 상태 확인
      4. CORS 설정 확인
      5. SSL 인증서 확인
      6. 로그 모니터링
      7. 성능 테스트

11. 도메인 연결
    - 가비아 DNS 설정:
      1. DNS 관리 페이지 접속
      2. A 레코드 추가:
         * 호스트: @
         * 값: Render 프론트엔드 IP
      3. CNAME 레코드 추가:
         * 호스트: api
         * 값: Render 백엔드 도메인
      4. SSL 인증서 적용 확인
      5. DNS 전파 확인
